# 製作規模探索 — 圖庫、示範與智能選擇

> 系統如何幫助使用者理解自己想做甚麼、
> 瀏覽真實例子，並在承諾預算或啟動代理人之前，
> 選出合適的製作規模。

---

## 問題

```text
現時流程（新手很容易混亂）：
  建立 Project →「選 A–J 範本」→「設定時長」

使用者想：
  「我唔知範本 E 代表乜」
  「30 秒夠唔夠講清我個 idea？」
  「$50 嘅 production 同 $200 嘅 production 分別係乜？」
  「可唔可以先睇下系統實際會做出啲乜？」

他們真正需要：
  先瀏覽 → 先獲得靈感 → 先理解規模 → 然後先配置
```

---

## 解法：Discovery Hub（新增一個頂層區段）

```text
更新後導航：

Side Nav：
  ○ Dashboard
  ○ Discover    ← 新增（Galleries + Showcases + Templates）
  ○ Projects
  ○ Productions
  ○ Agents
  ○ Settings
```

## 規模偵測如何運作（後端邏輯）

當使用者描述 idea（或選擇 showcase）時，系統會推斷規模：

```python
# 根據使用者輸入推斷規模（廉價的 LLM 呼叫）

async def infer_production_scale(user_input: str) -> ScaleRecommendation:
    """使用者在 Smart Selection Wizard 輸入描述時呼叫。"""
    
    response = await llm.classify(
        system="""You are a video production consultant. 
        Given a user's description of what they want to create,
        recommend the best production scale and template.
        
        Consider: platform (social=short, web=medium, cinema=long),
        content type (ad=15-60s, explainer=1-3min, narrative=3min+),
        complexity (talking head=simple, VFX=complex, multi-scene=long),
        audience expectations, and budget sensitivity.""",
        
        user_input=user_input,
        
        output_schema={
            "recommended_template": "A-J",
            "duration_range": {"min_seconds": int, "max_seconds": int},
            "estimated_cost": {"low": float, "mid": float, "high": float},
            "estimated_time_minutes": int,
            "agents_needed": int,
            "aspect_ratio": "16:9 | 9:16 | 1:1",
            "reasoning": str,
            "alternatives": [{
                "template": str,
                "why": str,
                "tradeoff": str
            }]
        }
    )
    
    return ScaleRecommendation(**response)
```

## 更新後的使用者旅程（加入 Discovery）

```text
之前（容易混亂）：
  建立 Project →「選 A–J 範本」→ 混亂 → 放棄

之後（有引導）：
  建立 Project → 瀏覽 Discover Hub → 睇到真實例子 →
  →「就係呢個！我想做類似呢個」→
  → 按 [Use as Template] → 以合適規模預填 brief →
  → 調整細節 → Launch

或者（AI 引導）：
  建立 Project → 用一句英文描述 idea →
  → AI 推薦：「你需要 15 秒 Hook，原因係…」→
  → 顯示 3 個該規模的例子 →
  → 使用者揀一個 → 預填 brief → Launch

或者（社群靈感）：
  瀏覽 Community Gallery → 睇到趨勢內容 →
  →「我想做類似呢個」→
  → 按 [Remix] → Fork 他嘅 brief 再調整 → Launch
```

## 系統如何知道製作規模：總結

| 方法 | 做法 | 使用者投入 |
|--------|-------------|-------------|
| **Template Carousel** | 視覺卡片（時長／成本／例子）瀏覽 | 低 — 主要靠看 |
| **Scale Guide** | 各規模並排對照表 | 低 — 讀完選 |
| **Showcase Gallery** | 真實成品 + 完整 metadata | 幾乎零 — 睇就得 |
| **Smart Wizard** | 用一句描述 → AI 推薦規模 | 極少 — 打一句 |
| **Community Feed** | 睇其他人做嘅 → fork/remix | 幾乎零 — 瀏覽 |
| **Inspiration Board** | 儲存參考 → 系統由集合推斷風格／規模 | 自然 — 儲存喜好 |
| **From Showcase** | 按「Use This as Template」→ 完全按該作品設定預填 | 一鍵 |

所有路徑都會去到同一個目的地：**一個已預填好範本、時長、畫面比例與預算的 Draft Production** — 讓使用者再微調後啟動。
