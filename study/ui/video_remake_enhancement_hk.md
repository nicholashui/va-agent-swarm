# 影片重製與增強 — 上載舊影片，取得 AI 改善方案

> 使用者上載一段既有影片。系統會從所有品質維度進行分析，提出一份詳細的改善方案；並在使用者確認後，使用完整的 114 個代理人陣容生成強化版本。

---

## 概念：「重製模式」

```text
使用者旅程：

  上載舊影片 → 系統分析（免費）→ 顯示改善方案 →
  → 使用者檢視／調整 → 確認 → 代理人重新生成改良版本 →
  → 並排對照 → 交付

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   這不只是「升頻／放大」。                                     │
│   這是由 114 個代理人進行的「完整創意重構」：                   │
│                                                                 │
│   • 劇本／敘事改進（節奏、開場鉤子、結構）                      │
│   • 視覺品質（解像度、構圖、色彩、風格）                        │
│   • 音訊品質（旁白、音樂、音效、混音）                          │
│   • 成效優化（留存、互動、ROAS）                                │
│   • 平台優化（格式、字幕、縮圖）                                │
│   • 合規檢查（無障礙、C2PA、權利）                               │
│                                                                 │
│   使用者在花錢之前先得到一份方案。                               │
│   他們可以全數接受、只選特定改善項，或自行調整。                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```text

═══════════════════════════════════════════════════════════════════

步驟 1：匯入
  • 擷取影片 metadata（時長、解像度、codec、fps）
  • 擷取音軌
  • 抽樣關鍵幀（每 1 秒 + 場景切換）
  • 音訊轉錄（Whisper v4）
  • 場景／剪接偵測（PySceneDetect）

  ┌──────────────────────────────────────────────────────────┐

  │ NoveltyAgent → 陳腔濫調偵測、原創性分數                   │
  └──────────────────────────────────────────────────────────┘

步驟 3：方案生成（PlannerAgent）
  • 彙總所有發現
  • 以 影響 × 可行性 排序
  • 估算每項改善成本
  • 分組為預算層級（快速／建議／全面）
  • 生成具體行動項並指派代理人

步驟 4：呈現給使用者
  • 以核取方塊呈現改善方案
  • 使用者挑選要修正的項目
  • 確認預算
  • → 然後才啟動製作（代理人開始工作）
```

## 如何連接到既有系統

```text
重製模式使用相同的代理人、相同的協調編排、相同的基礎設施。

唯一的差別：
  • 一般製作：由 brief 開始（文字 → 影片）
  • 重製製作：由既有影片開始（影片 → 分析 → 方案 → 更好的影片）

分析階段是一個特殊的前期製作步驟，用來：
  1. 從既有影片建立「反向 brief」
  2. 找出目前品質與最佳品質之間的差距
  3. 生成針對性的 DAG（只修正真正壞掉的部分）
  4. 使用者確認後，就像任何製作一樣啟動

一旦啟動，就會走相同流程：
  PlannerAgent → OrchestratorAgent → RouterAgent → Craft Agents → Gates → Delivery
```

## 重製模式的關鍵 UX 原則

| 原則 | 實作方式 |
|-----------|---------------|
| **免費分析** | 評估永遠不收費 — 只在生成時收費 |
| **透明評分** | 清楚展示哪裡有問題，以及可改善到甚麼程度 |
| **使用者掌控** | 核取方塊 — 使用者決定套用哪些改善 |
| **預算分級** | 快速／建議／全面 — 不強迫全有或全無 |
| **前後對照證據** | 並排對照 + 同步播放 |
| **非破壞式** | 永不修改或刪除原始影片 |
| **可迭代** | 第一次重製後可按「繼續迭代」再精修 |
| **可分享的方案** | 先匯出 PDF 給團隊／客戶批准再花錢 |
