This section discusses an automated system called the Quality Assurance Agent. This system checks quality before video and audio are delivered, acting like a super-professional inspector that meticulously examines every frame, sound, and technical detail, catching issues humans might miss. It uses Netflix's quality measurement technology that has won Emmy Awards.

Let's break down the key concepts in simple terms. VMAF (Video Multimethod Assessment Fusion) is like a high-level "quality score" for videos—similar to a credit score measuring financial health. It rates videos from 0 to 100, with higher scores indicating better quality. Netflix developed it to ensure videos look great whether viewed on a phone or a 60-inch TV.

Multi-Pass Validation is like proofreading an article multiple times, each pass targeting different issues. The system checks videos in six independent stages:
1. Basic specs, like resolution and frame rate;
2. Visual issues, such as flickering or blurriness;
3. Audio quality and synchronization;
4. Continuity errors, like props appearing or disappearing;
5. Overall viewing quality;
6. Final delivery checklist.

Object Tracking uses AI, similar to facial recognition, to follow objects in the video—like people, props, or vehicles—ensuring they don't mysteriously change or vanish between shots.

Now, the main topics:

1. **Visual Quality Analysis**: Detects issues like flickering lights, compression artifacts (blocky images), banding (obvious color steps), and incorrect focus.

2. **Audio Quality Control**: Checks for sync issues between audio and video, unwanted noise like hums or hisses, clipping distortion, and ensures volume levels are just right across platforms.

3. **Continuity Checking**: Uses AI to track character clothing, props, and environmental details, ensuring consistency throughout scenes.

4. **Technical Compliance**: Verifies videos meet industry standards, such as resolution, color space, frame rate, and metadata requirements.

5. **Automated Corrections**: Automatically fixes common issues like adjusting volume levels; complex problems are flagged for human review.

Key points summary: The six-stage validation system checks every aspect of video and audio before delivery; VMAF scores over 95 exceed human perception thresholds (streaming platforms typically target 84-92); AI-driven object tracking catches continuity errors as objects move through scenes; automated fixes handle common issues like re-encoding or de-noising without human intervention; platform-specific optimization ensures content looks great on phones, TVs, and cinema screens; real-time monitoring catches issues during production, not just at the end; industry-standard tools like VMAF, YOLO, FFmpeg, and librosa provide pro-level analysis.

Key numbers and data:
- **Quality benchmarks**: Top cinema (VMAF >95 for 4K versions); high-end streaming (90-95); standard broadcast (85-90); YouTube/Facebook (84-92 for user content).
- **Audio standards**: Dynamic range—cinema at -23 LUFS, streaming at -16 LUFS; noise floor—dialogue under -60dB, ambient under -80dB.
- **Performance**: VMAF-CUDA acceleration processes 4K videos 37x faster; optimized algorithms improve speed by 2x; quality predictions have 95% confidence intervals.

# Quality Assurance Agent - The Perfectionist

<#0.5#>

## System Overview

The Quality Assurance (QA) Agent serves as the final safeguard for production quality, employing a multi-pass validation framework to ensure that every frame, audio sample, and metadata element adheres to professional delivery standards. This agent integrates industry-standard tools, such as Netflix's Emmy-winning Video Multimethod Assessment Fusion (VMAF) algorithm, with advanced machine learning models to identify issues that traditional quality control processes might overlook.

<#0.5#>

### Core Architecture

#### Multi-Pass Validation Pipeline System

The system features a six-stage validation pipeline, with each stage conducting specialized analysis:

- **Pass 1**: Technical specification validation, including resolution, frame rate, color space, and bit depth.
- **Pass 2**: Visual artifact detection via computer vision algorithms.
- **Pass 3**: Audio quality analysis and synchronization validation.
- **Pass 4**: Content continuity scanning using object detection.
- **Pass 5**: Perceptual quality assessment with advanced metrics.
- **Pass 6**: Delivery compliance verification.

#### Advanced Visual Artifact Detection Engine

The system incorporates Netflix's Contrast Aware Multiscale Banding Index (CAMBI) for banding artifact detection, along with custom algorithms for the following:

- **Flickering Detection**: Analyzes frame-to-frame brightness variance using temporal filtering to identify irregular luminance patterns from lighting glitches or encoding artifacts.
- **Noise Pattern Analysis**: Applies spectral analysis to detect inconsistent grain structures, indicating issues with film emulation or denoising algorithms.
- **Compression Artifact Identification**: Uses frequency domain analysis to identify blocking, mosquito noise, and ringing artifacts that degrade visual quality.
- **Motion Blur Validation**: Evaluates edge coherence during camera movements to confirm appropriate shutter angle simulation and motion rendering.
- **Focus Quality Assessment**: Employs gradient-based sharpness metrics to verify depth of field rendering and detect unintended softness.

#### Comprehensive Audio Quality Analyzer

Built on the librosa Python library for audio analysis, the system extracts features such as Mel-frequency cepstral coefficients (MFCCs), spectral contrast, and chroma features:

- **Sync Validation Engine**: Applies temporal alignment algorithms to detect audio-video synchronization issues, requiring reference points present in both streams.
- **Phase Coherence Analysis**: Performs stereo and surround sound phase analysis in the frequency domain to identify out-of-phase conditions causing audio cancellation.
- **Spectral Quality Assessment**: Conducts Short-Time Fourier Transform (STFT) analysis to detect unwanted frequencies, clipping, and spectral anomalies.
- **Loudness Compliance Monitoring**: Verifies content against broadcast standards using Loudness Units Full Scale (LUFS) measurements.
- **Dynamic Range Analysis**: Assesses audio dynamics to ensure compliance with distribution requirements while preserving artistic intent.

#### Intelligent Continuity Error Scanner

This module leverages You Only Look Once (YOLO) object detection and tracking algorithms to assign and maintain object identifiers across frames for:

- **Character Appearance Consistency**: Uses convolutional neural networks (CNNs) to evaluate character clothing, accessories, and appearance across shots.
- **Prop and Set Continuity**: Tracks object positions and presence in scenes, assigning unique identifiers to detected objects in subsequent frames.
- **Environmental Coherence**: Analyzes lighting conditions, shadow directions, and atmospheric elements to detect temporal and spatial inconsistencies.
- **Temporal Logic Validation**: Cross-references time-dependent elements, such as clocks, watches, and sun positions, to identify illogical progressions.

#### Perceptual Quality Measurement System

VMAF, developed by Netflix in collaboration with the University of Southern California (USC), Université de Nantes, and the University of Texas, forms the core perceptual quality metric:

- **Multi-Method Fusion Approach**: Integrates multiple elementary quality metrics via Support Vector Machine (SVM) regression, with the fusion algorithm assigning optimal weights to leverage each metric's strengths.
- **Resolution-Specific Models**: Applies tailored VMAF models for various viewing conditions, including 1080p High-Definition Television (HDTV) at a 3H viewing distance in living rooms, mobile phone screens, and 4K displays at a 5H viewing distance.
- **Confidence Intervals**: Delivers 95% confidence intervals with each VMAF score to indicate assessment reliability.
- **Content-Adaptive Analysis**: Supports genre-specific calibration (e.g., sports, animation, dark scenes) and custom model training for specialized content.

#### Technical Specification Validator

This component performs comprehensive metadata analysis using industry-standard tools:

- **Format Compliance**: Utilizes ExifTool to read timed metadata from MOV, MP4, M2TS, and AVI videos, supporting various output formats.
- **Color Space Validation**: Confirms High Dynamic Range (HDR; Rec. 2020) and Standard Dynamic Range (SDR; Rec. 709) color primaries, gamma curves, and bit depth.
- **Frame Rate Accuracy**: Verifies precise frame timing and detects dropped frames that could impair playback.
- **Audio Channel Mapping**: Validates multi-channel configurations, including Dolby Atmos, 5.1, and stereo downmix compatibility.

## Advanced Implementation Details

<#0.5#>

### Visual Analysis Engine Deep Dive

#### Flickering Detection Algorithm

The system uses a temporal analysis pipeline with a sliding window to examine luminance histograms across consecutive frames, detecting irregular patterns indicative of lighting or encoding issues. Detection thresholds adjust dynamically by content type—for instance, action sequences with rapid changes use higher sensitivity than static dialogue scenes.

#### Noise Consistency Analysis

Employing the Mel frequency scale to model human perception, the system decomposes grain patterns spectrally to ensure consistent film emulation across shots. Statistical noise variance measures identify inconsistencies from uneven denoising or varying noise models.

#### Compression Artifact Detection

The engine applies frequency domain techniques for artifact identification:

- **Blocking Detection**: Employs edge detection in 8x8 pixel blocks to spot Discrete Cosine Transform (DCT) compression artifacts.
- **Mosquito Noise Identification**: Analyzes high-frequency components around edges to detect pre-echo artifacts in lossy compression.
- **Banding Detection**: Integrates Netflix's CAMBI algorithm to identify contouring in gradients.

<#0.5#>

### Audio Analysis Deep Dive

#### Synchronization Measurement

Audio-video sync issues are categorized as audio leading or lagging video, each demanding distinct corrections. The system uses cross-correlation between audio waveforms and visual events for frame-level precision.

#### Spectral Analysis Framework

The engine extracts features including MFCCs for speech, chroma for harmonics, and spectral contrast for amplitude distribution. These enable detection of:

- **Clipping and Distortion**: Amplitude distribution statistics flag peaks exceeding 0 dBFS.
- **Unwanted Frequencies**: Spectral analysis identifies power line hum (50/60 Hz), air conditioning noise, and environmental contaminants.
- **Phase Issues**: Channel correlation reveals phase cancellation in stereo or surround setups.

### Object Tracking and Continuity Analysis

#### YOLO-Based Continuity System

YOLO's real-time detection combines with tracking algorithms to sustain object identities across frames, following a tracking-by-detection paradigm as in Multiple Object Tracking (MOT). Detections inform tracking while preserving consistent identifiers.

#### Advanced Tracking Implementation

Algorithms like ByteTrack apply to detection models for video-wide tracking, using supervision libraries for computer vision development. Tracking logs capture:

- Object appearance timestamps and durations.
- Spatial positions across frames.
- Consistency metrics for color, texture, and scale.
- Occlusion handling and re-identification.

### VMAF Quality Assessment Integration

#### Multi-Model Architecture

VMAF fuses elementary metrics via SVM regression for scores from 0–100, where >95 exceeds human perception thresholds. Platforms like YouTube and Facebook aim for 84–92 in user-generated content.

#### Implementation Strategy

VMAF v2.0+ includes fixed-point and x86 SIMD optimizations (AVX2, AVX-512) for 2x speed gains, with extensible APIs. Integration occurs as an FFmpeg filter for processing compressed bitstreams without full decoding.

### Framework: Netflix VMAF

#### Implementation Approach

VMAF is an Emmy-winning perceptual video quality algorithm from Netflix, offering a C library (`libvmaf`) and Python wrapper.

#### Core Architecture

```python
# Production-ready VMAF integration
from vmaf import VmafConfig, VmafFeatureExtractor
import subprocess
import json

class VMAfQualityAnalyzer:
    def __init__(self):
        # VMAF v3.0 with optimizations and bug fixes
        self.vmaf_path = "/usr/local/bin/vmaf"
        self.models = {
            'default': 'vmaf_v0.json',
            '4k': 'vmaf_4k_v0.json',
            'mobile': 'vmaf_v0.1neg.json'
        }

    def analyze_quality(self, reference_path, distorted_path, model='default'):
        cmd = [
            'vmaf',
            '--reference', reference_path,
            '--distorted', distorted_path,
            '--model', f'version={self.models[model]}',
            '--json'
        ]
        result = subprocess.run(cmd, capture_output=True, text=True)
        return json.loads(result.stdout)
```

#### Advanced Features Integration

CAMBI detects banding artifacts. VMAF integrates as an FFmpeg filter via `./configure --enable-libvmaf`.

**Pros**:
- Fixed-point and x86 SIMD optimizations (AVX2, AVX-512) yield 2x speedups.
- Outperforms SSIM, PSNR-HVS, and VQM-VFD.
- Includes PSNR, PSNR-HVS, SSIM, MS-SSIM, and CIEDE2000.

**Cons**:
- Trained datasets are proprietary.
- Requires full-reference video.
- High computational demands for 4K.

#### CUDA Acceleration Integration

VMAF-CUDA, from NVIDIA-Netflix collaboration, is in VMAF 2.0+ and FFmpeg v6.0+, offering up to 4x throughput and 37x lower 4K latency.

## Audio Quality Analysis Implementation

<#0.5#>

### Framework: librosa

#### Implementation Approach

Librosa is a Python package for music and audio analysis, supporting file loading, spectrograms, feature extraction, and visualization.

#### Core Architecture

```python
import librosa
import numpy as np
from scipy import signal
import matplotlib.pyplot as plt

class AudioQualityAnalyzer:
    def __init__(self, sr=22050):
        self.sr = sr
        self.feature_extractors = {
            'mfcc': self.extract_mfcc,
            'spectral_contrast': self.extract_spectral_contrast,
            'chroma': self.extract_chroma_features,
            'tonnetz': self.extract_tonnetz
        }

    def analyze_sync(self, video_path, audio_path):
        """Audio/video sync analysis using cross-correlation"""
        # Load audio streams
        audio_video, _ = librosa.load(video_path, sr=self.sr)
        audio_separate, _ = librosa.load(audio_path, sr=self.sr)

        # Cross-correlation for sync detection
        correlation = signal.correlate(audio_video, audio_separate, mode='full')
        delay = correlation.argmax() - len(audio_separate) + 1
        return delay / self.sr  # Return delay in seconds

    def extract_mfcc(self, audio_path, n_mfcc=13):
        """Extract MFCC features for speech analysis"""
        y, sr = librosa.load(audio_path, sr=self.sr)
        mfccs = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=n_mfcc)
        return mfccs
```

#### Advanced Feature Extraction

Librosa extracts MFCCs, spectral contrast, tonnetz, and more for classification, genre recognition, and retrieval. The Mel scale models human frequency perception.

**Pros**:
- Handles diverse formats; integrates with NumPy and SciPy for large datasets.
- Supports beat tracking and tempo estimation.
- Separates harmonic and percussive components.

**Cons**:
- Older soundfile versions (<0.11) fallback to audioread for MP3.
- Limited real-time support for high sample rates.
- Requires dependencies for certain formats.

## Object Detection and Continuity Tracking Implementation

<#0.5#>

### Framework: Ultralytics YOLO

#### Implementation Approach

Ultralytics trackers output standard detections with object IDs, facilitating video stream tracking and analytics.

#### Core Architecture

```python
from ultralytics import YOLO
import cv2
from collections import defaultdict
import numpy as np

class ContinuityTracker:
    def __init__(self, model_path="yolo11n.pt"):
        self.model = YOLO(model_path)
        self.track_history = defaultdict(list)
        self.continuity_errors = []

    def track_objects(self, video_path, classes_of_interest=None):
        """Track objects across video frames for continuity analysis"""
        cap = cv2.VideoCapture(video_path)
        frame_count = 0

        while cap.isOpened():
            ret, frame = cap.read()
            if not ret:
                break

            # Run tracking with persistence
            results = self.model.track(
                frame, 
                persist=True, 
                classes=classes_of_interest,
                conf=0.5  # Adjusted for clarity
            )

            # Analyze continuity for each tracked object
            self.analyze_frame_continuity(results, frame_count)
            frame_count += 1

        cap.release()
        return self.continuity_errors

    def analyze_frame_continuity(self, results, frame_num):
        """Analyze object appearance consistency"""
        if results[0].boxes is not None:
            for box in results[0].boxes:
                if box.id is not None:
                    track_id = int(box.id)
                    bbox = box.xyxy[0].cpu().numpy()
                    confidence = box.conf[0].cpu().numpy()
                    class_id = int(box.cls[0])

                    # Store tracking history
                    self.track_history[track_id].append({
                        'frame': frame_num,
                        'bbox': bbox,
                        'confidence': confidence,
                        'class': class_id
                    })

                    # Check for continuity issues
                    self.check_continuity_errors(track_id, frame_num)
```

#### Multi-Algorithm Support

Supports BoT-SORT (default; `botsort.yaml`) and ByteTrack (`bytetrack.yaml`). Integrates with segmentation and pose estimation for flexible tracking.

**Pros**:
- Real-time efficiency without accuracy loss.
- Supports detection, segmentation, pose, tracking, and classification.
- Simple Python API and CLI; customizable with trained models.

**Cons**:
- Challenges with fast-moving or crossing objects.
- Relies on general models; tracker-specific training in development.

## Video Processing and Transcoding Implementation

### Framework: FFmpeg

#### Implementation Approach

FFmpeg is an open-source suite for multimedia handling, including transcoding, editing, scaling, effects, and compliance.

#### Core Architecture

```python
import subprocess
import json
import os
from concurrent.futures import ThreadPoolExecutor

class VideoProcessor:
    def __init__(self):
        self.ffmpeg_path = "ffmpeg"
        self.supported_codecs = self.get_supported_codecs()

    def validate_technical_specs(self, video_path):
        """Validate video meets technical specifications"""
        probe_cmd = [
            'ffprobe', '-v', 'quiet', '-print_format', 'json',
            '-show_format', '-show_streams', video_path
        ]

        result = subprocess.run(probe_cmd, capture_output=True, text=True)
        metadata = json.loads(result.stdout)

        # Extract video stream properties
        video_stream = next(
            (s for s in metadata['streams'] if s['codec_type'] == 'video'), 
            None
        )

        if video_stream:
            return self.check_specifications(video_stream)

    def apply_quality_corrections(self, input_path, output_path, corrections):
        """Apply automated quality corrections"""
        cmd = [self.ffmpeg_path, '-i', input_path]

        # Build filter chain based on corrections
        filters = []

        if 'deinterlace' in corrections:
            filters.append('yadif')

        if 'noise_reduction' in corrections:
            filters.append('hqdn3d')

        if 'color_correction' in corrections:
            filters.append(f"eq=contrast={corrections['contrast']}:brightness={corrections['brightness']}")

        if filters:
            cmd.extend(['-vf', ','.join(filters)])

        # Add encoding parameters
        cmd.extend([
            '-c:v', 'libx264', '-preset', 'slow', '-crf', '18',
            '-c:a', 'aac', '-b:a', '320k',
            output_path
        ])

        return subprocess.run(cmd, capture_output=True, text=True)
```

#### Hardware Acceleration

FFmpeg supports NVIDIA GPU acceleration via the Video Codec SDK for encoding/decoding, enabling high-performance pipelines.

**Pros**:
- >100 codecs; supports x86, PPC, ARM, etc.
- Comprehensive features for multimedia processing.
- Libraries like libavcodec, libavformat, and libavfilter.

**Cons**:
- Potential patent issues in some regions.
- Command-line complexity for advanced tasks.
- Lossy encoding degrades quality; use judiciously.

## Metadata Validation and Extraction Implementation

<#0.5#>

### Framework: ExifTool

#### Implementation Approach

ExifTool is a Perl-based command-line tool for Image::ExifTool, enabling metadata reading/writing via simple commands like `exiftool FILE`.

#### Core Architecture

```python
import subprocess
import json
import re
from datetime import datetime

class MetadataValidator:
    def __init__(self):
        self.exiftool_path = "exiftool"
        self.validation_rules = self.load_validation_rules()

    def extract_comprehensive_metadata(self, file_path):
        """Extract all metadata from video/image file"""
        cmd = [
            self.exiftool_path, '-json', '-a', '-G', '-s',
            '-ee',  # Extract embedded data
            file_path
        ]

        result = subprocess.run(cmd, capture_output=True, text=True)

        if result.returncode == 0:
            metadata = json.loads(result.stdout)[0]
            return self.process_metadata(metadata)
        else:
            raise Exception(f"ExifTool error: {result.stderr}")

    def validate_delivery_compliance(self, metadata):
        """Validate metadata against delivery specifications"""
        validation_results = {
            'compliant': True,
            'issues': [],
            'warnings': []
        }

        # Check technical specifications
```

## Audio Quality Analysis Implementation

### Framework: librosa

<#0.5#>

#### Implementation Approach

Librosa is a Python package for music and audio analysis, supporting file loading, spectrograms, feature extraction, and visualization.

<#0.5#>

#### Core Architecture

```python
import librosa
import numpy as np
from scipy import signal
import matplotlib.pyplot as plt

class AudioQualityAnalyzer:
    def __init__(self, sr=22050):
        self.sr = sr
        self.feature_extractors = {
            'mfcc': self.extract_mfcc,
            'spectral_contrast': self.extract_spectral_contrast,
            'chroma': self.extract_chroma_features,
            'tonnetz': self.extract_tonnetz
        }

    def analyze_sync(self, video_path, audio_path):
        """Audio/video sync analysis using cross-correlation"""
        # Load audio streams
        audio_video, _ = librosa.load(video_path, sr=self.sr)
        audio_separate, _ = librosa.load(audio_path, sr=self.sr)

        # Cross-correlation for sync detection
        correlation = signal.correlate(audio_video, audio_separate, mode='full')
        delay = correlation.argmax() - len(audio_separate) + 1
        return delay / self.sr  # Return delay in seconds

    def extract_mfcc(self, audio_path, n_mfcc=13):
        """Extract MFCC features for speech analysis"""
        y, sr = librosa.load(audio_path, sr=self.sr)
        mfccs = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=n_mfcc)
        return mfccs
```

Librosa extracts MFCCs, spectral contrast, tonnetz, and more for classification, genre recognition, and retrieval. The Mel scale models human frequency perception.

## Object Detection and Continuity Tracking Implementation

<#0.5#>

### Framework: Ultralytics YOLO

<#0.5#>

#### Implementation Approach

Ultralytics trackers output standard detections with object IDs, facilitating video stream tracking and analytics.

<#0.5#>

#### Core Architecture

```python
from ultralytics import YOLO
import cv2
from collections import defaultdict
import numpy as np

class ContinuityTracker:
    def __init__(self, model_path="yolo11n.pt"):
        self.model = YOLO(model_path)
        self.track_history = defaultdict(list)
        self.continuity_errors = []

    def track_objects(self, video_path, classes_of_interest=None):
        """Track objects across video frames for continuity analysis"""
        cap = cv2.VideoCapture(video_path)
        frame_count = 0

        while cap.isOpened():
            ret, frame = cap.read()
            if not ret:
                break

            # Run tracking with persistence
            results = self.model.track(
                frame, 
                persist=True, 
                classes=classes_of_interest,
                conf=0.5  # Adjusted for clarity
            )

            # Analyze continuity for each tracked object
            self.analyze_frame_continuity(results, frame_count)
            frame_count += 1

        cap.release()
        return self.continuity_errors

    def analyze_frame_continuity(self, results, frame_num):
        """Analyze object appearance consistency"""
        if results[0].boxes is not None:
            for box in results[0].boxes:
                if box.id is not None:
                    track_id = int(box.id)
                    bbox = box.xyxy[0].cpu().numpy()
                    confidence = box.conf[0].cpu().numpy()
                    class_id = int(box.cls[0])

                    # Store tracking history
                    self.track_history[track_id].append({
                        'frame': frame_num,
                        'bbox': bbox,
                        'confidence': confidence,
                        'class': class_id
                    })

                    # Check for continuity issues
                    self.check_continuity_errors(track_id, frame_num)
```

- Simple Python API and CLI; customizable with trained models.

**Cons**:

#### Implementation Approach

FFmpeg is an open-source suite for multimedia handling, including transcoding, editing, scaling, effects, and compliance.

<#0.5#>

#### Core Architecture

```python
import subprocess
import json
import os
from concurrent.futures import ThreadPoolExecutor

class VideoProcessor:
    def __init__(self):
        self.ffmpeg_path = "ffmpeg"
        self.supported_codecs = self.get_supported_codecs()

    def validate_technical_specs(self, video_path):
        """Validate video meets technical specifications"""
        probe_cmd = [
            'ffprobe', '-v', 'quiet', '-print_format', 'json',
            '-show_format', '-show_streams', video_path
        ]

        result = subprocess.run(probe_cmd, capture_output=True, text=True)
        metadata = json.loads(result.stdout)

        # Extract video stream properties
        video_stream = next(
            (s for s in metadata['streams'] if s['codec_type'] == 'video'), 
            None
        )

        if video_stream:
            return self.check_specifications(video_stream)

    def apply_quality_corrections(self, input_path, output_path, corrections):
        """Apply automated quality corrections"""
        cmd = [self.ffmpeg_path, '-i', input_path]

        # Build filter chain based on corrections
        filters = []

        if 'deinterlace' in corrections:
            filters.append('yadif')

        if 'noise_reduction' in corrections:
            filters.append('hqdn3d')

        if 'color_correction' in corrections:
            filters.append(f"eq=contrast={corrections['contrast']}:brightness={corrections['brightness']}")

        if filters:
            cmd.extend(['-vf', ','.join(filters)])

        # Add encoding parameters
        cmd.extend([
            '-c:v', 'libx264', '-preset', 'slow', '-crf', '18',
            '-c:a', 'aac', '-b:a', '320k',
            output_path
        ])

        return subprocess.run(cmd, capture_output=True, text=True)
```
