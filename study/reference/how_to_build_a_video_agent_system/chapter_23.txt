### Props Management Agent

A Props Management Agent is an AI-driven system designed to manage all physical props in movie production, from story-critical props (hero props) to background objects. It combines 3D modeling, physics simulation, historical validation, and production scheduling into an intelligent platform. This ensures props are accurate, available exactly when needed, and consistent across all scenes—making the production process much smoother.

### Key Concepts (Explained Simply)

**Universal Scene Description (USD)**: Think of it as a "master blueprint" system for 3D objects, like architects using layered blueprints where different people handle plumbing, wiring, and structure separately. USD allows multiple artists to work on the same prop simultaneously without conflicts.

**Bullet Physics**: A simulation engine that makes digital objects behave realistically, e.g., how a gun lands on a table, how a coffee cup shatters, or how debris scatters in an explosion.

**Procedural Generation**: No need to manually create a thousand different pens for background shots—the system automatically generates variations from a base design, like a factory producing similar but not identical products.

**Hero Props vs. Background Props**: Hero props are story-critical (e.g., Sarah's gun or evidence folder) with extreme detail. Background props (e.g., pens and paper) add realism but don't need individual scrutiny.

### Main Topics

1. **Advanced Asset Creation**: Creates highly accurate 3D models for hero props, auto-generates thousands of background objects, and manages multiple versions (e.g., new gun vs. old gun, or gun with evidence tag added).

2. **Physics and Interaction Simulation**: Simulates realistic object behavior, including weight, balance, and fragility. Builds destruction sequences for action scenes and material reactions (e.g., coffee sloshing or metal bending).

3. **Historical Accuracy Validation**: Connects to museum databases to verify era-appropriate props, auto-checks for cultural sensitivities, and flags anachronisms before production.

4. **Production Scheduling and Tracking**: Determines when each prop is needed, tracks status across scenes (e.g., gun in holster vs. on table), and manages dependencies (e.g., can't shoot Scene 12 without the evidence folder).

5. **Cross-Platform Deployment**: Exports props to software like Maya, Unreal Engine, or Unity; optimizes assets for each platform's requirements and automates technical conversions.

### Summary of Key Benefits

1. **Single Source of Truth**: All prop info in one intelligent system prevents continuity errors and communication mishaps.
2. **Massive Time Savings**: Procedural generation speeds up background props by 300% vs. manual methods.

#### Background Props Procedural System

The system applies advanced surface scattering for prop placement, isolating spawn areas, preventing floating via intersection checks, and using points for randomized prop instantiation with ratio controls.

```python
class BackgroundPropsScatterer:
    def __init__(self, surface_mesh, props_library, scene_context):
        self.surface = surface_mesh
        self.library = props_library
        self.context = scene_context

    def scatter_props(self, prop_type, density, constraints):
        # Analyze surface for valid placement zones
        valid_zones = self.analyze_surface_zones()

        # Apply scene-specific constraints
        valid_zones = self.apply_constraints(valid_zones, constraints)

        # Generate placement points using Poisson disk sampling
        placement_points = self.poisson_disk_sample(
            valid_zones, density, min_distance=self.calculate_min_distance()
        )

        # Assign props to points with variation
        scattered_props = []
        for point in placement_points:
            prop_variant = self.select_prop_variant(prop_type, point)
            prop_instance = self.instantiate_prop(prop_variant, point)
            scattered_props.append(prop_instance)

        return scattered_props

    def analyze_surface_zones(self):
        """Identify valid placement areas on surface"""
        zones = []

        # Check surface normals (avoid steep slopes)
        for face in self.surface.faces:
            normal = face.calculate_normal()
            if normal.dot(Vector3(0, 0, 1)) > 0.7:  # Less than 45° slope
                zones.append(face)

        # Remove zones too close to edges
        edge_distance_threshold = 0.05  # 5cm
        zones = [z for z in zones if self.distance_to_edge(z) > edge_distance_threshold]

        return zones

    def poisson_disk_sample(self, zones, density, min_distance):
        """Generate evenly distributed placement points"""
        points = []
        candidates = []

        # Initial random point
        if zones:
            initial_point = random.choice(zones).center
            points.append(initial_point)
            candidates.extend(self.generate_candidates(initial_point))

        while candidates and len(points) < density:
            candidate = candidates.pop(random.randint(0, len(candidates)-1))

            # Check minimum distance to existing points
            if all(candidate.distance(p) >= min_distance for p in points):
                points.append(candidate)
                candidates.extend(self.generate_candidates(candidate))

        return points
```

        """Calculate realistic moment of inertia tensor"""
        # Use component masses and distributions
        components = {
            'slide': {'mass': 0.285, 'center': Vector3(0.05, 0, 0.01)},

class AdvancedDestructionSystem:
    def __init__(self, physics_world, fracture_engine):
        self.physics_world = physics_world
        self.fracture_engine = fracture_engine
        self.destruction_presets = {}

    def setup_destructible_table(self, table_model):
        """Create breakable interrogation table with realistic failure points"""

        # Analyze table structure for weak points
        structural_analysis = {
            'material': 'institutional_steel',
            'weak_points': [
                {'location': 'weld_joints', 'failure_threshold': 5000},  # Newtons
                {'location': 'table_legs', 'failure_threshold': 8000},
                {'location': 'table_surface', 'failure_threshold': 12000}
            ],
            'failure_modes': ['bend', 'break', 'detach']
        }

        # Pre-fracture geometry using Voronoi patterns
        fractured_components = self.fracture_engine.voronoi_fracture(
            table_model,
            fracture_count=200,
            fracture_pattern='realistic_metal'
        )

        # Setup physics constraints holding pieces together
        constraints = []
        for i, component in enumerate(fractured_components):
            for j, neighbor in enumerate(component.get_neighbors()):
                constraint = self.create_breakable_constraint(
                    component, neighbor, structural_analysis['weak_points'][i % len(structural_analysis['weak_points'])]
                )
                constraints.append(constraint)

        return DestructibleObject(fractured_components, constraints)

    def simulate_table_break_sequence(self, impact_force, impact_point):
        """Simulate realistic table destruction sequence"""

        # Phase 1: Initial impact and stress distribution
        stress_propagation = self.calculate_stress_waves(impact_force, impact_point)

        # Phase 2: Constraint failure cascade
        failed_constraints = []
        for constraint in self.table_constraints:
            stress_at_constraint = stress_propagation.get_stress_at_point(constraint.position)

            if stress_at_constraint > constraint.failure_threshold:
                constraint.break()
                failed_constraints.append(constraint)

                # Trigger sound event
                self.trigger_audio_event('metal_break', constraint.position, stress_at_constraint)

        # Phase 3: Secondary collapse simulation
        remaining_structure = self.get_structure_without_constraints(failed_constraints)
        secondary_failures = self.simulate_structural_collapse(remaining_structure)

        # Phase 4: Debris physics
        debris_pieces = self.get_all_debris_pieces()
        for piece in debris_pieces:
            # Apply realistic material properties

                'restitution': 0.3,
                'sound_material': 'metal_debris'

### Phase 1: Core Foundation (Months 1-3)
- **USD Pipeline Setup**: Basic asset management and versioning.
- **PostgreSQL Database**: Props catalog and metadata.
- **Basic Blender Integration**: Manual creation and USD export.

<#0.5#>

### Phase 2: Physics and Procedural (Months 4-6)
- **Bullet Physics Integration**: Destruction and interaction simulation.
- **Elasticsearch Search**: Prop discovery and matching.
- **Houdini Engine**: Background prop generation.

<#0.5#>

### Phase 3: Advanced Features (Months 7-12)
- **Cross-Platform Deployment**: Exports to multiple engines.
- **AI-Powered Historical Validation**: Museum API integrations.
- **Production Scheduling**: Dependency and optimization management.

<#0.5#>

### Infrastructure Requirements

```yaml
# Docker Compose Stack
version: '3.8'
services:
  postgresql:
    image: postgres:15
    environment:
      POSTGRES_DB: props_management
    volumes:
      - postgres_data:/var/lib/postgresql/data

  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.0.0
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    volumes:
      - elastic_data:/usr/share/elasticsearch/data

  props_agent:
    build: .
    depends_on:
      - postgresql 
      - elasticsearch
    environment:
      - USD_LIBRARY_PATH=/usr/local/usd
      - BULLET_PHYSICS_PATH=/usr/local/bullet
```

<#0.5#>

## Critical Dependencies and Prerequisites

<#0.5#>

### System Requirements
- **Operating System**: Linux (preferred), macOS, Windows 10+.
- **Python**: 3.8+ with usd-core, PyBullet, Elasticsearch clients.
- **Hardware**: 16GB+ RAM, dedicated GPU for simulations.
- **Storage**: Fast SSD for caching and temps.

<#0.5#>

### Integration Challenges and Solutions

**USD Learning Curve**: 
- Start with basic layers; provide training/documentation.

**Houdini Licensing Costs**:
- Use Apprentice for dev; upgrade for production. Alternatives: Blender Geometry Nodes.

**Physics Simulation Performance**:
- Implement LOD/culling; use GPU (CUDA/OpenCL).

**Database Synchronization Complexity**:
- Monitor/rollback procedures; use PGSync for consistency.

This approach builds a robust, scalable system using proven open-source tools, with phased rollout to manage risks.
