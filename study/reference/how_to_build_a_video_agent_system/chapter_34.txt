## Archivematica Digital Preservation Architecture
Archivematica automates the preparation of digital objects for ingest into a repository and access system, following the ISO Open Archival Information System (OAIS) functional model. It employs a microservices design to process objects from ingest to access, creating Submission Information Packages (SIPs), Archival Information Packages (AIPs), and Dissemination Information Packages (DIPs).

<#0.5#>

### Core Functionality
Archivematica's microservices handle granular tasks on OAIS packages:
- **SIP**: Initial submission package with source files and metadata.
- **AIP**: Long-term archival package with preservation formats and integrity checks.
- **DIP**: Access package for dissemination in user-friendly formats.

### Comprehensive Workflow
Via the web interface, curators accept or reject transfers. Accepted transfers undergo initial analysis: checksum calculation, UUID assignment, virus scanning, format identification, and metadata extraction. This generates a SIP for further processing.

## LTO Tape Technology
Linear Tape-Open (LTO) is an open-standard magnetic tape format for backup, archiving, and transfer, developed in the late 1990s as an alternative to proprietary systems. It dominates the super tape market.

<#0.5#>

### Capacity and Specifications
As of 2025, LTO-10 offers 30 TB per cartridge (native). Cartridges use half-inch (12.65 mm) tape on a single reel. Capacity has grown from 100 GB in LTO-1 (2000) while maintaining physical size.

<#0.5#>

### Reliability and Lifespan
LTO tapes achieve a 30-year lifespan in controlled environments (temperature and humidity regulated). With a bit error rate of 1 in 10^19 bits—10,000 times better than hard disks and 1,000 times better than SSDs—reliability stems from advanced error correction codes (ECC) that detect and correct errors during writes.

<#0.5#>

### Linear Tape File System (LTFS)
LTFS (ISO 20919:2021) partitions tapes for direct file access: Partition 0 for index/metadata and Partition 1 for data. Supporting LTO-6+, it mimics disk or flash drive access, enabling drag-and-drop operations.

#### Operational Benefits
Mounted LTFS tapes appear as external drives, ensuring compatibility and straightforward file access. Periodic checksum validation maintains data integrity.

#### Best Practices
- Treat LTO-5+ tapes as removable disks via LTFS.
- Mount with `ltfs` for folder-based copying, avoiding command-line methods.
- Always unmount cleanly to prevent corruption.

<#0.5#>

### Environmental Requirements
Optimal conditions: 16-20°C (62-68°F) and 40-50% relative humidity. Use climate-controlled cases; acclimate tapes before use; avoid sunlight, heat, or moisture.

<#0.5#>

### Replacement Schedules
Replace tapes every 3-5 years for active use or 7-10 years for pure archiving to minimize data loss risks.

## PBCore Metadata Standard for Audiovisual Content
PBCore is an XML-based cataloging standard for audiovisual assets, developed in the early 2000s by U.S. public broadcasters. It supports description, data sharing, and management in databases or systems.

<#0.5#>

### Framework and Elements
Organized into fields for intellectual content (e.g., title, genre) and instantiations (physical/digital realizations). It extends Dublin Core with AV-specific details like duration, data rate, and aspect ratio.

<#0.5#>

### Asset and Instantiation Model
- **Asset**: Abstract intellectual content (e.g., a film's story).
- **Instantiation**: Specific formats/locations (e.g., LTO tape or cloud file), sharing asset metadata.

<#0.5#>

### Integration with Other Standards
PBCore complements PREMIS for preservation metadata, focusing on shared properties while deferring format-specific details. Combine via XML for technical AV descriptions.
