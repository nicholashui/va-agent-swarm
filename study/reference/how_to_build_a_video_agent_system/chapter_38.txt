# Press Kit Agent - The Media Distributor

<#0.5#>

## Overview

The Press Kit Agent is a comprehensive media distribution and public relations management system designed for film and entertainment marketing campaigns. This intelligent platform orchestrates the full press outreach lifecycle—from content creation to analytics reporting—to maximize media coverage and foster positive sentiment for film releases.

<#0.5#>

## Core Components

<#0.5#>

### Press Release Generator
An AI-powered writing tool that produces complete press releases adhering to Associated Press (AP) style guidelines. It includes all essential elements expected by journalists, integrates with Customer Relationship Management (CRM) systems, and maintains brand consistency across business units. Using advanced natural language processing (NLP), the system converts technical film production data into compelling, media-ready narratives.

<#0.5#>

### Media Asset Packager
A digital asset management system that stores and organizes diverse file types, including 3D content, R3D RAW files, ProRes videos, and 360-degree footage. It supports fast uploads and downloads in a secure, centralized repository. The system automatically organizes, compresses, and formats assets for various distribution channels while preserving quality to meet media outlet specifications.

<#0.5#>

### Interview Q&A System
An intelligent preparation tool that generates interview materials for talent and crew. It creates talking points, anticipates journalist questions, and provides strategic messaging guidance. The system frames key messages to resonate with journalists and target audiences, ensuring consistent communication across media interactions.

<#0.5#>

### Journalist Database
A CRM system that indexes and enriches hundreds of millions of data points using AI and dedicated research. It maintains current information for targeted pitches and improved coverage rates. The database automatically updates contact details, beat assignments, and journalist preferences.

<#0.5#>

### Embargo Manager
A timing control system that manages release schedules, tracks compliance, and prevents leaks. It integrates with legal frameworks and distribution networks to govern information flow throughout the campaign.

<#0.5#>

### Download Portal
A secure, authenticated web portal built with custom Django views for permission checks. It streams files from private folders not exposed via standard media URLs, implementing role-based access control and download tracking for accountability and analytics.

<#0.5#>

### Analytics Tracker
An advanced monitoring system using VADER (Valence Aware Dictionary and sEntiment Reasoner) for sentiment analysis. VADER, validated as the most accurate tool through extensive testing, enables real-time coverage monitoring, reach estimation, and sentiment evaluation across media channels.

<#0.5#>

### Content Creation and Management

#### AI-Powered Press Release Writing
The system adheres to journalism standards: compelling yet factual headlines, first paragraphs answering who/what/when/where/why/how, and properly attributed quotes. Assets receive metadata tags for searchability by upload date, publication date, title, themes, involved personnel, and departments. For film campaigns, it automates tagging of cast, scenes, locations, and production elements to facilitate database-wide analysis and retrieval.

Editors store content centrally while customizing and distributing images, audio, and video. Graphics and photography integrate seamlessly into film editing workflows. Assets auto-format for outlet needs, from high-resolution print to web-optimized social media.

#### Intelligent Journalist Targeting
The system segments journalists by circulation, audience demographics, and past coverage sentiment, identifying tier-one reporters with positive competitor coverage. This supports strategic messaging to boost favorable outcomes.

```python
def generate_press_release(template_type, film_details, target_audience):
    """
    Advanced press release generation with context-aware prompting.
    """
    prompts = {
        "casting": f"""
        Write a professional press release for {film_details['genre']} film "{film_details['title']}" 
        announcing {film_details['actor']} joining the cast.

        CONTEXT:
        - Film Genre: {film_details['genre']}
        - Target Audience: {target_audience}
        - Director: {film_details['director']}
        - Actor Credentials: {film_details['actor_background']}
        - Character: {film_details['character_description']}

        REQUIREMENTS:
        - Lead with compelling hook related to actor's star power
        - Include specific actor credentials (awards, previous successful films)
        - Director quote emphasizing collaboration and actor's unique contribution
        - Character description (intriguing but spoiler-free)
        - Production timeline and location if available
        - Industry-standard contact information

        STYLE:
        - AP Style formatting
        - Entertainment industry tone (enthusiastic but professional)
        - 300-400 words
        - Include quotation marks around all spoken quotes
        - Use past tense for confirmed information, future tense for upcoming events
        """
    }

    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[
            {"role": "system", "content": "You are an expert entertainment industry publicist with 15 years experience writing film press releases for major studios. You understand both journalism standards and entertainment industry expectations."},
            {"role": "user", "content": prompts[template_type]}
        ],
        max_tokens=800,
        temperature=0.7,  # Balanced creativity with consistency
        presence_penalty=1,  # Encourage diverse language
        frequency_penalty=1   # Reduce repetitive phrases
    )

    return response['choices'][0]['message']['content']
```

```python
def comprehensive_fact_check(text, verified_data, industry_database):
    """
    Multi-layer fact verification system.
    """
    import spacy
    from datetime import datetime
    import re

    nlp = spacy.load("en_core_web_sm")
    doc = nlp(text)

    errors = []
    warnings = []

    # Verify dates
    for ent in doc.ents:
        if ent.label_ == "DATE":
            date_text = ent.text
            try:
                parsed_date = parse_date(date_text)
                if parsed_date not in verified_data.get("approved_dates", []):
                    if is_future_date(parsed_date) and not in_production_schedule(parsed_date):
                        errors.append(f"Invalid production date: {date_text}")
                    elif is_past_date(parsed_date) and not in_historical_records(parsed_date):
                        warnings.append(f"Unverified historical date: {date_text}")
            except ValueError:
                errors.append(f"Malformed date: {date_text}")

    # Verify people and credentials
    for ent in doc.ents:
        if ent.label_ == "PERSON":
            person_name = ent.text
            if person_name not in verified_data.get("confirmed_cast_crew", []):
                if not verify_person_in_industry_db(person_name, industry_database):
                    errors.append(f"Unverified person in industry: {person_name}")
                else:
                    warnings.append(f"Person not in confirmed cast/crew list: {person_name}")

    # Verify award claims
    award_patterns = [
        r"Oscar winner", r"Academy Award winner", r"Emmy winner", 
        r"Golden Globe winner", r"BAFTA winner"
    ]

    for pattern in award_patterns:
        matches = re.finditer(pattern, text, re.IGNORECASE)
        for match in matches:
            context = extract_context_around_match(text, match)
            if not verify_award_claim(context, verified_data.get("awards", {})):
                errors.append(f"Unverified award claim: {context}")

    # Verify technical details
    technical_terms = extract_technical_terms(text)
    for term in technical_terms:
        if term in ["IMAX", "Dolby Atmos", "4K", "HDR"] and not verify_technical_spec(term, verified_data):
            warnings.append(f"Unverified technical specification: {term}")

    return {
        "errors": errors,
        "warnings": warnings,
        "confidence_score": calculate_confidence_score(errors, warnings),
        "verification_report": generate_detailed_report(text, doc, verified_data)
    }
```

```python
def generate_comprehensive_interview_materials(subject_info, film_details, campaign_context):

            "weight": 20,

        },

        },
        "thematic_content": {
            "weight": 20,
            "focus": "Film themes, social relevance, audience impact"
        },
        "industry_context": {
            "weight": 10,
            "focus": "Career development, industry trends, future projects"
        },
        "personal_connection": {
            "weight": 10,
            "focus": "Why this project, personal growth, life experiences"
        }
    }

    all_questions = {}

    for category, details in question_categories.items():
        prompt = f"""
        Generate {int(20 * details['weight'])} interview questions for {subject_info['name']} 
        ({subject_info['role']}) about the film "{film_details['title']}".

        CATEGORY FOCUS: {details['focus']}

        CONTEXT:
        - Film Genre: {film_details['genre']}
        - Character: {subject_info.get('character_name', 'N/A')}
        - Director: {film_details['director']}
        - Release Strategy: {campaign_context['release_strategy']}
        - Target Audience: {campaign_context['target_audience']}

        QUESTION REQUIREMENTS:
        - Engaging and conversational tone
        - Avoid generic questions that could apply to any film
        - Include follow-up potential
        - Range from light/fun to substantive/serious
        - Avoid spoilers but allow for intriguing hints
        - Consider journalist audience (trade vs mainstream vs niche)

        Format as numbered list without answers.
        """

        response = openai.ChatCompletion.create(
            model="gpt-4",
            messages=[
                {"role": "system", "content": "You are an experienced entertainment journalist who knows how to ask compelling questions that reveal interesting insights while respecting promotional boundaries."},
                {"role": "user", "content": prompt}
            ],
            max_tokens=600,
            temperature=0.8  # Higher creativity for diverse questions
        )

        questions = parse_numbered_list(response['choices'][0]['message']['content'])
        all_questions[category] = questions

    return all_questions
```

```python
    import mimetypes
    from wsgiref.util import FileWrapper

    class SecureMediaView(LoginRequiredMixin, DetailView):
        """
        Secure media file serving with comprehensive access control.
        """
        def get(self, request, *args, **kwargs):
            file_id = kwargs.get('file_id')

            if not self.check_user_permissions(request.user, file_id):
                raise PermissionDenied("Insufficient permissions for this content")

            file_info = self.get_file_info(file_id)

            if not file_info or not os.path.exists(file_info['path']):
                raise Http404("File not found")

            if not self.check_embargo_status(file_info, request.user):
                return render(request, 'embargo_restricted.html', {
                    'embargo_lift_date': file_info['embargo_date']
                })

            self.log_download_activity(request.user, file_info)

            return self.serve_secure_file(file_info, request)

        def check_user_permissions(self, user, file_id):
            """
            Multi-level permission checking.
            """
            if not user.is_authenticated:
                return False

            if not user.groups.filter(name='Approved Press').exists():
                return False

            file_permissions = self.get_file_permissions(file_id)
            user_clearance_level = self.get_user_clearance_level(user)

            return user_clearance_level >= file_permissions['required_level']

        def serve_secure_file(self, file_info, request):
            """
            Optimized secure file serving with multiple backend support.
            """
            file_path = file_info['path']

            if settings.USE_X_SENDFILE:
                response = HttpResponse()

                response['Content-Type'] = mimetypes.guess_type(file_path)[0]
                response['Content-Disposition'] = f'attachment; filename="{file_info["name"]}"'
                return response

            elif settings.USE_NGINX_ACCEL:
                response = HttpResponse()
                response['X-Accel-Redirect'] = f"/protected-files/{file_info['name']}"
                response['Content-Type'] = mimetypes.guess_type(file_path)[0]
                response['Content-Disposition'] = f'attachment; filename="{file_info["name"]}"'
                return response

            else:
                try:
                    response = FileResponse(
                        open(file_path, 'rb'),
                        content_type=mimetypes.guess_type(file_path)[0]
                    )
                    response['Content-Disposition'] = f'attachment; filename="{file_info["name"]}"'
                    return response
                except IOError:
                    raise Http404("File could not be read")
```

```python
                "transformer_sentiment": self.news_sentiment(article_text[:512]),
                "entertainment_sentiment": self.entertainment_sentiment(article_text[:512]),
                "metadata": article_metadata
            }

            )

                "final_sentiment": context_adjusted_score,
                "confidence": self.calculate_confidence(results),
                "breakdown": results,
                "key_phrases": self.extract_sentiment_drivers(article_text)
            }

        def calculate_ensemble_sentiment(self, results):
            """
            Weighted ensemble of multiple sentiment models.
            """
            vader_sentiment = results["vader_scores"]["compound"]

                results["transformer_sentiment"][0]

                vader_sentiment * weights["vader"] +
                transformer_score * weights["transformer"] +
```

Delegates serving to backends with permission checks.

**Implementation Approach:**
```python
# Secure download portal with Django-Filer
from filer.models import File
from django.contrib.auth.decorators import login_required
from django.http import HttpResponse

class SecureMediaView(LoginRequiredMixin, DetailView):
    def get(self, request, *args, **kwargs):
        file_id = kwargs.get('file_id')

        if not self.check_user_permissions(request.user, file_id):
            raise PermissionDenied()

        file_obj = File.objects.get(id=file_id)
        if not self.check_embargo_status(file_obj, request.user):
            return self.embargo_restricted_response()

        response = HttpResponse()
        response['X-Accel-Redirect'] = f'/protected-files/{file_obj.name}'
        response['Content-Type'] = file_obj.mime_type

        self.log_download_activity(request.user, file_obj)

        return response
```

**Production Optimization:**
- X-Accel-Redirect offloads from Django.

**Alternative: django-private-storage**
Protects uploads with login; uses storage APIs.

**Security Features:**
- Proxies via views; avoids direct byte serving.

**Pros:**
- Performance via webserver delegation.
- Seamless admin integration.

**Cons:**
- Needs server config.
- Some alternatives unmaintained.

```python
# VADER sentiment analysis with real-time monitoring
from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer
import streamlit as st
import plotly.express as px

class RealTimeSentimentMonitor:
    def __init__(self):
        self.analyzer = SentimentIntensityAnalyzer()

    def analyze_coverage_sentiment(self, articles):
        sentiments = []
        for article in articles:
            scores = self.analyzer.polarity_scores(article['content'])
            sentiments.append({
                'article_id': article['id'],
                'compound': scores['compound'],
                'positive': scores['pos'],
                'negative': scores['neg'],
                'neutral': scores['neu'],
                'outlet': article['outlet'],
                'timestamp': article['published_at']
            })
        return sentiments

    @st.cache_data(ttl=300)  # Cache for 5 minutes
    def create_sentiment_dashboard(self, sentiments_df):
        fig = px.scatter(
            sentiments_df,
            x='timestamp',
            y='compound',
            color='outlet',
            size='positive',
            title='Real-Time Sentiment Analysis'
        )
        return fig
```
