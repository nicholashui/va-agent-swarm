## What to Build

- **Physics-Based Stunt Simulator**: A real-time collision detection and multi-physics simulation system based on the Bullet Physics engine. It leverages rigid body dynamics, soft body simulation, and collision detection for realistic stunt scenarios. Key input parameters include friction and restitution coefficients to model material interactions accurately.

- **Safety Validation System**: An intelligent framework that assesses stunt feasibility using biomechanical constraints, G-force calculations, and injury probability matrices. It runs Monte Carlo simulations to evaluate thousands of variations per sequence, ensuring compliance with human safety limits in extreme scenarios.

- **Action Sequence Choreographer**: A deep learning framework that synthesizes high-quality animations from unstructured motion data without manual labeling. It supports skills such as ball dribbling, professional maneuvers, shooting, catching, avoidance, and multiple locomotion modes. The system uses a multi-dimensional phase space from full-body motion data to cluster animations and improve similarity measures over traditional motion spaces.

- **Fight Choreography Generator**: A deep learning framework for controllable martial arts movements from raw motion capture data. It employs neural network-based animation layering to blend and edit unaligned motions. The system includes a database of motion patterns, timing, and biomechanical constraints for styles including boxing, karate, kung fu, jujitsu, and mixed martial arts.

- **Vehicle Chase Planner**: A modular API built on the CARLA autonomous driving simulator. It handles vehicle dynamics, sensor suites, environmental conditions, and control of static/dynamic actors. Features include traffic generation, weather simulation, and advanced models for suspension dynamics, tire friction, aerodynamics, and engine performance.

- **Fall/Explosion Simulator with Injury Risk Calculator**: A module integrating collision detection and soft body dynamics with medical biomechanics data. It computes impact forces, deformation patterns, and injury probabilities using trauma databases and forensic reconstruction methodologies.

- **Camera Placement Optimizer**: A cinematography system that analyzes 3D action sequences to optimize camera positions, movements, and transitions. It balances visual clarity, dramatic impact, continuity, and filming constraints while preventing collisions with environment geometry.

## What It Does

<#0.5#>

### Primary Functions

**Comprehensive Action Design**: This agent serves as a full stunt design ecosystem. It analyzes screenplay requirements to generate physically accurate, cinematically compelling sequences. Inputs include narrative beats, character motivations, environmental constraints, and safety parameters, producing choreographies that balance realism and drama.

**Physics Validation and Safety Assurance**: Powered by the Bullet Physics engine's collision detection and multi-physics capabilities (proven in games and visual effects), the system verifies sequences against physics principles and human limits. It conducts real-time biomechanical analysis, including G-forces, impact velocities, and joint strain, to avoid injury risks.

**Intelligent Choreography Generation**: Advanced deep learning frameworks synthesize dynamic movements from unstructured motion data. For fights, it applies codebook matching to resolve ambiguities and produce high-quality controllers from motion capture data.

**Vehicle Dynamics Simulation**: Using CARLA's Unreal Engine foundation and ASAM OpenDRIVE road standards, with Python and C++ APIs, the system creates realistic chases. It incorporates sensor models (cameras, LIDAR, RADAR) and privileged data like semantic segmentation and depth maps.

**Advanced Pre-visualization**: The agent generates sequences with industry-standard rendering engines, allowing directors and coordinators to visualize, analyze, and refine stunts pre-production. This approach cuts costs, reduces risks, and supports iterative creativity.

<#0.5#>

### Detailed Operational Workflows

**Motion Capture Integration and Processing**: The system processes recordings from the CMU Graphics Lab Motion Capture Database, covering walking, dancing, sports, and activities by over 140 subjects. It uses enhanced BVH format conversions with T-poses aligned to the positive Z-axis for MotionBuilder compatibility, enabling seamless integration with animation pipelines.

**Physics Engine Integration**: Bullet Physics SDK drives real-time collision detection and multi-physics simulation for VR, games, visual effects, robotics, and machine learning. It combines the rigid-body API for dynamics with the bullet-featherstone plugin for multibody simulations based on Featherstone's algorithms, optimized for articulated systems like human kinematic chains.

**Character Animation and Control**: The system blends physics results with keyframed animation for realistic ragdoll effects. Using nodes like `Set All Bodies Simulate Physics` and `Set All Bodies Below Physics Blend Weight`, it activates Physics Asset bodies on Skeletal Meshes recursively, ensuring lifelike limb responses during impacts or falls.

**Vehicle Physics and Chase Choreography**: CARLA's Traffic Manager orchestrates non-primary vehicles to mimic urban environments. Integration with Scenic, a probabilistic programming language, allows custom scenarios with actor placement and behavior control for choreographed chases.
