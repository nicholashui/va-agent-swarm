# 建立中央 LLM API 使用和成本儀表板應用程序

## 項目名稱建議
**LLMUsageHub** 或 **MultiLLM 儀表板** 或 **API 成本中心** 或 **LLM 支出追蹤器**

## 一、專案概況
建立一個**網頁應用程式**，提供**單一中央視圖**，用於追蹤使用者的**所有** LLM API 帳户的使用情況、成本、餘額、支出和代幣消耗。

用户目前擁有以下帳户：
- x.ai（Grok API）
- Poe  
- MiniMax  
- 基米（Moonshot AI）
- OpenRouter  
……以及許多其他人。

該應用程式應該讓用户添加一次 API 金鑰，然後查看**在一個漂亮的儀表板中聚合的所有內容** - 每月總支出、剩餘積分、每個提供者的細分、圖表、趨勢、警報等。

**參考/靈感**：
受到 **[cc-switch](https://github.com/farion1231/cc-switch)** （用於管理 Claude Code / Codex / Gemini CLI 的 LLM 提供者的流行桌面工具）的啟發。此 Web 應用程式**純粹專注於跨直接個人 API 金鑰的使用/成本分析**，作為 cc-switch 的僅使用伴侶，但作為 Web 應用程式。

## 2. 核心目標
- 一個統一的地方來監控**所有** LLM 支出和使用情況。
- API 金鑰的安全性、僅本機儲存（從不傳送到任何伺服器）。
- 自動或按需取得使用/計費資料。
- 歷史追蹤+視覺化。
- 極其可擴展——易於添加新的提供者。
- 漂亮、現代的使用者介面類似 cc-switch。

## 3. 主要功能（必備）

### 供應商管理
- 新增/編輯/刪除帳户：名稱、提供者類型（預設）、API 金鑰、基本 URL（用於自訂端點）、註解。
- 為盡可能多的提供者預先建置**預設**（請參閲第 4 節）。
- 支援每個提供者多個帳户。
- 一鍵「全部刷新」和個別刷新按鈕。

### 使用和餘額獲取
- 首選可用的 **官方 API**（例如 `/usage`、`/billing`、`/balance`、`/v1/token_plan/remains` 等）。
- 後備選項：
  - 手動輸入目前使用情況/餘額。
  - Web 儀表板抓取（如果需要，使用 Playwright，最後的手段）。
- 後台自動刷新（可設定間隔）+手動刷新。
- 將完整歷史快照儲存在本機資料庫中。

### 儀表板使用者介面
- **概述頁面**：
  - 預計美元支出總額（今天/本月/所有時間）。
  - 總剩餘積分/餘額（如果可能，標準化）。
  - 活躍提供者的數量 + 快速狀態。
- **提供者卡**（網格或清單）：
  - 名稱 + 徽標（如果有）。
  - 當前餘額/剩餘積分。
  - 本月支出+趨勢指標。
  - 最後更新的時間戳記。
- **圖表**：
  - 支出趨勢（折線圖 - 每日/每週）。
  - 按提供者細分的成本（餅圖）。
  - 按模型劃分的令牌使用情況（條）。
  - 使用熱圖或日曆視圖。
- **詳細表格**：
  - 每個提供者的使用歷史記錄。
  - 模型級細分。
- **警報**：
  - 低餘額警告（可配置閾值）。
  - 每日高支出通知。

### 成本計算
- 主要模型的內建定價表（輸入/輸出代幣 → 美元）。
- 允許使用者覆蓋每個型號的定價。
- 即使提供者僅報告代幣，也會顯示估計的美元。

### 數據持久化和導出
- 所有歷史使用快照的本機 **SQLite** 資料庫。
- 將完整資料匯出為 CSV 或 JSON。

### 安全與使用者體驗
- API 金鑰在本機**加密**儲存（Fernet 對稱加密）。
- 深色/淺色主題（預設為深色，匹配現代人工智慧工具）。
- 可從本機存取基於瀏覽器的 UI。
- 初始設定後完全離線。
- 響應靈敏、乾淨、專業的使用者介面。

### 錦上添花（第二階段）
- AI 驅動的見解（「本月您在 Kimi 上花費了 68% - 考慮將繁重的任務切換到 Groq」）。
- 導入/匯出配置（包括可能的 cc-switch 導入）。
- 每個模型的成本預測。
- 可選的代理/路由器模式（如 LiteLLM 或 cc-switch），因此應用程式還可以記錄實際 API 呼叫的使用情況。

## 5. 技術棧

### Backend
- **框架**：帶有 FastAPI 的 Python 3.11+
- **API**：OpenAPI 3.1（從 FastAPI 自動生成，可在 /docs 瀏覽）
- **資料庫**：帶有 SQLAlchemy ORM 的 SQLite
- **HTTP 用户端**：httpx（非同步）
- **安全性**：使用 Fernet 加密技術靜態加密 API 金鑰
- **後台任務**：FastAPI 後台任務 + APScheduler

### API 設計（OpenAPI）
- 所有 CRUD 操作的 RESTful 端點
- 自動產生 OpenAPI 模式
- 透過 Swagger UI 的互動式 API 文檔，位於 /docs
- ReDoc 替代方案位於 /redoc

### Frontend
- **框架**：React 18 + TypeScript
- **建置工具**：Vite
- **樣式**：TailwindCSS + shadcn/ui
- **圖表**：重新圖表
- **狀態管理**：Zustand
- **HTTP 用户端**：Axios 或 fetch API

### Architecture
- **Web 應用程式**（非桌面）- 在瀏覽器中本機執行
- 後端以本機伺服器運作（localhost:8000）
- 由 FastAPI 靜態檔案或單獨的 Vite 開發伺服器提供服務的前端
- 100% 本地 — 除非稍後明確添加，否則不會進行雲端同步

## 6. 開發階段（建議）
1. 專案設定（FastAPI 後端 + React 前端 + SQLite）。
2. 提供者管理+安全密鑰儲存。
3. 核心使用fetcher系統（抽象介面）。
4. 實作 5-6 個高優先級提供者（xAI、Poe、OpenRouter、MiniMax、Kimi、OpenAI）。
5. 儀表板 UI + 圖表。
6. 新增剩餘的提供者+定價表。
7. 後台刷新、警報、匯出、潤飾。
8. 測試+文件。

## 7. 可交付成果
- 完整的源代碼，帶有出色的註釋和自述文件。
- 關於**如何新增提供者**（新的 Python 模組 + 定價配置）的明確説明。
- 設定使用 FastAPI + React 在本機運行的腳本。
- 樣本資料/測試模式。
- 許可證：MIT（或任何使用者喜歡的）。
