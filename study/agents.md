## 2. Common Structure of an AI Agent

Every agent in the swarm — regardless of category — is built on the same skeleton. The structure is derived from the column headers of the master roster combined with the architecture patterns (§1), critique protocol (§6), and universal success-criteria framework (§5) of the source document.

| # | Component | Purpose | Mechanism / Reference |
|---|---|---|---|
| 1 | **Identity** | Stable, unique handle used for routing, logging, and provenance signing | e.g. `DirectorAgent`, `EditorAgent` (kebab-case ID + semantic version) |
| 2 | **Responsibility (Scope)** | Single-sentence definition of what the agent owns end-to-end | Per-row "Responsibility" column; mirrors a human craft role |
| 3 | **Knowledge Distillation Source** | The licensed / consented corpora and expert-interview data the agent is trained or RAG-grounded on | Award archives (DGA, ASC, ACE…), academic papers, paid expert interviews, peer-reviewed journals |
| 4 | **Tool Access** | External APIs, generators, validators, and DCC bridges the agent can call | Sora / Veo / Runway / Kling, ElevenLabs, Sync.so, Hedra, Resolve / Nuke / AE via MCP, ReAct pattern (Yao et al. 2022) |
| 5 | **Architecture Pattern** | The reasoning / learning loop that powers the agent | Self-Refine (Madaan 2023), Reflexion (Shinn 2023), RLAIF / Constitutional AI (Bai 2022), Multi-agent debate (Du 2023), LLM-as-Judge (Zheng 2023), Pairwise preference / Arena, Tool-use / ReAct |
| 6 | **Memory** | Episodic + long-term project memory for context recall and learning | Vector DB / MemGPT-style store accessed via MemoryAgent (§2.9.1) |
| 7 | **Constitution / Rubric** | Written, role-specific scoring guide the agent self-checks against | e.g. Murch's Rule of Six (Editor), 12 principles (Animator), Save-the-Cat beats (Screenwriter), MT-Bench rubrics |
| 8 | **Self-Quality Criteria (L1 + L2)** | Pre-handoff self-test on structured spec and craft rubric | L1: JSON schema + tool validators (codec, LUFS, aspect, length) — pass = 100%. L2: LLM-as-judge with role-specific constitution — pass ≥85/100 |
| 9 | **Surpass-Human Signal (L3)** | Pre-registered measurement that proves the agent has reached or exceeded a credentialed human professional | Pairwise blind preference (Arena ≥55%), benchmark dominance (VBench, USMLE, CFA L3, MQM), certification pass, lower 90-day defect rate, higher novelty at equal quality |
| 10 | **Critique Inbox (Accepts From / How)** | Channel through which other agents (and HiTL) can critique this agent's outputs | Shared `CritiqueMessage` JSON bus (§6) — severity = blocker / major / minor / nit; triggers Self-Refine / Reflexion loop; disputes go to JudgeAgent (multi-agent debate, Du 2023) |
| 11 | **Critique Outbox (Comments On)** | The set of peer agents whose work this agent is qualified to review | Listed per agent in the master roster; messages emitted on the same `CritiqueMessage` bus and appended to C2PA provenance |
| 12 | **HiTL Escalation Path** | When a human must be brought into the loop (regulation / ethics / novelty) | Consent for voice & likeness (SAG-AFTRA AI rider, EU AI Act Art. 50), final legal sign-off, MPA rating, festival eligibility, crisis comms, cross-cultural sensitivity review |
| 13 | **Provenance & Signing** | Every artifact the agent emits is cryptographically signed and the chain is verified by downstream agents | C2PA spec — accepted critiques are appended to the artifact's chain for full audit |
| 14 | **Continuous Distillation Hook** | How the agent keeps learning from real practitioners post-deployment | Bootstrap on licensed corpora → consented expert interviews → live RLAIF (DPO / KTO) → award-rubric grounding → adversarial red-team → 30 / 60 / 90-day reality-check metrics (§7) |
| 15 | **Phase-Gate Compliance** | The agent only emits to the next phase after L1, L2, **and** L3 all pass, verified by GateKeeperAgent | Universal 3-layer success-criteria framework (§5): Spec → Rubric → Preference |

### CritiqueMessage Schema (shared by all agents)

```json
{
  "critique_id": "uuid",
  "from_agent": "EditorAgent",
  "to_agent": "DirectorAgent",
  "artifact_ref": "shot_42_take_3.mp4",
  "severity": "blocker | major | minor | nit",
  "category": "pacing | continuity | accuracy | compliance | accessibility | brand | craft",
  "evidence": ["timecode 00:01:14 — held 1.4s past the cut point per genre prior"],
  "suggested_action": "trim 1.0s; re-evaluate hold",
  "rubric_reference": "Murch Rule of Six §3",
  "must_resolve_before": "phase_4_review"
}
```

> **Acceptance rules** (verbatim from §6 of source): blocker halts the DAG; major triggers a Self-Refine / Reflexion loop (≤3 iterations); minor / nit is logged for the next training cycle (RLAIF reward signal); two-agent disputes go to JudgeAgent multi-agent debate, then HiTL if unresolved; every accepted critique is appended to C2PA provenance.
