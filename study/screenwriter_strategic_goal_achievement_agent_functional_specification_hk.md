

## 策略目標達成框架實踐論證：「編劇」深度探索

**章節目標：** 透過完整的「編劇」案例研究，示範如何使用六階段自我質疑框架將模糊的想法轉化為具體的、可操作的計劃。

**學習重點：**
- 如何從表面答案挖掘核心動機
- 如何將抽象概念轉化為具體行動
- 如何辨識並突破思維盲點
- 如何建立可持續的執行策略

**開源框架支援：**
- **深度挖掘：** 使用[Five Whys Framework](https://github.com/lean-startup-circle/five-whys)進行根本原因分析
- **行動轉化：**採用[Getting Things Done (GTD)](https://github.com/gtd-methodology/gtd-tools)方法論
- **思維突破：**應用[Design Thinking Toolkit](https://github.com/designthinkingtools/toolkit)的移情圖
- **執行策略：**整合[OKR Framework](https://github.com/7geese/okr-framework)目標管理系統

**用法：** 您可以按照此演示，將“劇本寫作”替換為您的任何目標，並應用相同的提問技巧進行深入探索。

## 行動框架：從模糊的想法到明確的行動



### **第一階段：動機與目的**－為什麼要從事編劇？

**原始表面答案：** “編劇是表達我內心感受的一種方式……治愈自己，或者挑戰現實。”

**問題診斷：** 這個答案太抽象，太安全，缺乏個人色彩和情感分量。

**深挖過程：**

**🔧 應用框架：[Five Whys Root Cause Analysis](https://github.com/lean-startup-circle/five-whys)**

**問題：** 為什麼我想寫劇本？表面原因是什麼？更深層的原因是什麼？

**初步回答：** 我想表達內心的感受，療癒自己，挑戰現實。

**五個為什麼實施：**
1. **為什麼1：**為什麼我要表達我內心的感受？ → 因為我有話要説
2. **為什麼2：**為什麼我有話要説？ → 因為我經歷過痛苦
3. **為什麼3：**為什麼痛苦需要表達？ → 因為我不想讓別人經歷同樣的孤獨
4. **為什麼4：**為什麼我要關心別人的孤獨？ → 因為我深知孤獨的滋味
5. **為什麼5：**為什麼透過劇本而不是其他方式？ → 因為故事能觸動人心最深處

**後續問題：** 這個答案可以接受嗎？夠具體嗎？我能感受到情感上的共鳴嗎？

**進一步的問題：**「治癒我自己」—我想治癒什麼？是童年創傷嗎？心碎的痛苦？對世界的憤怒？請生動地描述一個讓我意識到「我必須寫劇本」的具體事件或經歷。

**深化答案範例：**
「記得有一次，深夜看完一部家庭破裂的電影，我突然淚流滿面。因為這讓我想起了自己家裡的冷漠——爸爸總是忙得無話可説，媽媽總是抱怨，而我卻被夾在中間，不知道該怎麼辦。那天晚上我想，如果我能寫一個故事來講述這種痛苦，那不僅是為了我自己，也可能幫助其他孤獨的人。

**問：**如果我不寫劇本，十年後我會有什麼遺憾？

**深化答案範例：**
「我會後悔自己有自己想講的故事，但卻把這些感受埋藏在內心深處，因為我害怕失敗，害怕被嘲笑，最後成為一個沉默的人。我會後悔看到別人寫類似主題的劇本，並想‘我也想過這樣寫’，但它永遠只是‘想到’。最大的遺憾就是沒有嘗試用自己的聲音去改變一個人的世界觀。」

**答案品質評估：**
✅ **特異性** - 具有特定的觸發事件和場景描述
✅ **情感真實性** - 能感受到真正的痛苦與使命感
✅ **可操作性** - 明確定義寫作方向與主題
✅ **內部一致性** - 與個人經驗和價值觀保持一致
✅ **深度** - 從興趣提升到任務級別

**關鍵突破：**從「表達」到「見證」的概念轉變，賦予目標更深層的意義。



### **第二階段：受眾與背景**－我們為誰寫作？

**原面回答：** “首先是為了我自己……其次是為了觀眾，比如年輕人、苦苦掙扎的上班族……”

**問題診斷：**受眾描述過於寬泛，缺乏具體的人物形象，無法引導創作方向。

**深挖過程：**

**🔧 應用框架：[Design Thinking Empathy Map](https://github.com/designthinkingtools/empathy-map)**

**問題：** 我的觀眾是誰？我可以具體描述一個「典型觀眾」成員嗎？

**初步回答：** 年輕人、上班族。

**移情圖建構：**
- **説：**“我的工作沒有意義”，“我不知道我的人生方向”
- **認為：** 懷疑自己的選擇，擔心未來，渴望改變
- **是：** 加班、玩手機、假裝很高興、避免深入交談
- **感覺：**孤獨、焦慮、困惑、被困住的感覺

**使用者角色創建：**
使用[Persona Template](https://github.com/uxtools/persona-template)建立特定的角色圖像

**後續問題：** 這個答案太廣泛了。哪些年輕人？幾歲了？他們面臨什麼困難？

**深化答案範例：**
「我的觀眾是阿欣，25歲的AE廣告公司，每天加班到晚上10點，回到家已經筋疲力盡，無法做自己喜歡的事情。她曾經想成為一名設計師，但現在只是沒完沒了地為客户修改PPT。每天晚上睡覺前，她都會問自己：『這就是我的生活嗎？』我想為她修改PPT。每天晚上睡覺前，她都會問自己：『這就是我的生活嗎？』我想為她寫作，因為我曾經是她希望我想為她寫作後的一些劇本，『我想為她寫作，因為我曾經是她想要改變了。

**問題：**為什麼這個時候寫「現在」？是什麼讓這個時機如此特別？

**深化答案範例：**
「因為現在的社會氛圍——經濟不景氣、房價暴漲、擔心人工智能取代人類工作——讓很多人覺得自己的努力毫無意義。我覺得現在正是需要故事來提醒大家，即使在一個荒誕的世界裡，我們仍然可以選擇如何生活。這個時候不寫，以後就覺得沒什麼意義了。”

**答案品質評估：**
✅ **具體** - 擁有具體的人物（阿信）與生活細節
✅ **情感連結** - 與觀眾建立同理心
✅ **Timing Insight** - 了解當前社會環境的影響
✅ **需要清晰** - 知道觀眾需要什麼類型的內容

**關鍵突破點：** 從“所有人”縮小到“某個特定的人”，給創造一個明確的對話對象。



### **第 3 階段：方法與限制** — 如何寫？有什麼限制？

**原始表面反應：**“我可能會使用非線性敍事......為什麼不使用線性敍事？因為如果故事太可預測......”

**問題診斷：**方法選擇缺乏深入的概念支持，限制理解不夠清晰，可能影響執行效果。

**深挖過程：**

**🔧 應用框架：[Design Constraints Canvas](https://github.com/designthinking/constraints-canvas)**

**問題：** 我打算如何寫作？哪些方法對我來説是自然的？

**初始答案：** 非線性敍事。

**約束畫布分析：**
- **必須具備：** 情感真實、性格深度
- **應有：**結構創新，視覺衝擊力
- **可能有：** 多個結局，互動元素
- **不會：** 過度暴力、刻板印象

**方法選擇框架：**
使用[Creative Process Framework](https://github.com/creative-process/methodology)評估方法選擇

**後續問題：** 為什麼是非線性的？只是因為「令人驚訝」那麼簡單嗎？還是有更深層的原因？

**深化答案範例：**
「我用非線性敍事，因為我相信記憶本身是碎片化的。當我們回憶起人生中的重要時刻時，並不是按順序播放的，而是某種氣味、某個短語，突然把你拉回到多年前。我想透過非線性結構來模擬這種『突然了解過去』的感覺——觀眾只有在故事進行到一半時才會明白為什麼主角在第一個場景會發生「那樣會發生「」「為什麼」。 ’”

**問題：** 我必須遵守哪些限制？為什麼這些限制無法被打破呢？

**深化答案範例：**
「我的限制是時間——我只有90分鐘的電影長度（或一個系列的6集），所以我不能無限地擴展次要情節。這種限制其實是一種祝福，因為它迫使我做出選擇：哪些情節是真正必要的？哪些角色真正推動了故事？限制讓我的劇本更加精緻。此外，我有道德限制——我不想僅僅為了震撼價值而添加過多的暴力或剝削性內容。我希望我的劇本，即使是黑暗的，也能保留人性的尊嚴。 」

**答案品質評估：**
✅ **概念支持** - 非線性敍事有明確的哲學基礎
✅ **限制意識** - 明確時間與道德限制
✅ **創意轉型** - 將限制視為創意催化劑
✅ **個人風格** - 方法與個人價值觀一致

**關鍵突破點：** 從“技術選擇”提升到“概念表達”，賦予方法靈魂。



### **第四階段：情感期望**－我想要感受什麼？

**最初的表面反應：** “完成劇本後，我希望感受到一種‘解放’的感覺……還有一點自豪感……”

**問題診斷：** 情緒描述過於抽象，缺乏感官細節，難以作為真正的動力來源。

**深入研究過程：**

**🔧 應用框架：[Emotional Design Toolkit](https://github.com/emotional-design/toolkit)**

**問題：** 完成劇本後，我想體驗什麼情感？你能用感官術語來描述它們嗎？

**初步反應：** 解放，自豪。

**情緒映射技術：**
- **身體感覺：** 肩膀放鬆，呼吸順暢，胸部温暖
- **情緒強度：** 1-10 評級系統
- **持續時間：** 即時感覺與長期滿意度
- **觸發因素：** 完成的時刻、他人的認可、自我肯定

**軀體體驗框架：**
使用[Body Awareness Tools](https://github.com/somatic-tools/awareness) 追蹤身體感覺

**後續問題：** 「解放」是什麼感覺？在我身體的什麼地方？什麼温度、質地？

**深化回應範例：**
「『解放』對我來説就像在炎熱的夏天脱掉一件濕透的T卹——清涼、輕鬆、可以深呼吸。我感覺到肩膀上的重量卸下了，胸口的壓力消失了。這種感覺會持續幾天，每次回想起‘我完成了’，我都會情不自禁地微笑。驕傲不是浮誇的，而是一種安靜的滿足——我坐在窗邊，看著劇本文件，心想『這是我的』。 」

**問題：**我在寫作過程中希望感受到什麼？我願意忍受哪些負面情緒？

**深化回應範例：**
「我希望在這個過程中感受到流動——時間消失了，我完全沉浸其中的狀態。我願意忍受挫折（例如不知道如何正確地寫某個場景）、自我懷疑（『這太糟糕了嗎？』），甚至疲憊（為了趕在最後期限前通宵達旦）。但我知道這些負面情緒是旅程的一部分，也是創作的重要部分。我會與它們共存，而不是迴避它們。 」

**回應品質評估：**
✅ **特定感官** - 用身體感覺描述情緒（脱掉 T 恤的涼爽感）
✅ **預體驗** - 能夠想像和感受目標情緒
✅ **過程準備** - 應對負面情緒的心理準備
✅ **永續性** - 知道情緒會持續多久

**關鍵突破：** 將抽象的字詞轉化為具體的身體感覺，讓情感變得有形。



### **第 5 階段：執行與影響** — 我希望觀眾有何反應？

**原始表面答案：**“我希望觀眾感到受到啟發......我希望他們討論，分享，甚至爭論......”

**問題診斷：**預期影響過於模糊，缺乏可觀察的成功指標，難以評估效果。

**深入研究過程：**

**🔧 應用框架：[Impact Measurement Canvas](https://github.com/impact-measurement/canvas)**

**問題：** 我希望觀眾收到什麼？他們的生活將會發生哪些具體的改變？

**最初的答案：** 靈感，舒適。

**影響畫布設計：**
- **輸入：** 劇本內容、情感投入、時間與精力
- **活動：**觀看、討論、分享、反思
- **輸出：** 觀眾回饋、社群媒體互動、口碑傳播
- **結果：** 行為改變、情緒療癒、關係改善
- **影響力：** 提高社會意識，促進文化對話

**行為改變框架：**
使用[Behavior Change Wheel](https://github.com/behavior-change/wheel)分析受眾行為轉變

**後續問題：** 「靈感」是什麼意思？他們會怎麼想？他們會做什麼？

**深化答案範例：**
「我希望觀眾看完後會想，『我不是唯一一個有這種想法的人。』」例如，如果劇本是關於孤獨的，他們可能會意識到周圍的很多人也很孤獨，只是戴著面具。然後他們可能會主動與朋友聊天，或對陌生人表現出更多的温柔。我並不是要求他們做大事，但我希望他們在日常生活中獲得更多的認識。 」

**問題：** 什麼反應會告訴我「成功了」？

**深化答案範例：**
“如果有人看完後給我發信息，説‘這個角色就是我’，我就知道這是成功的。如果有人在社群媒體上寫長文分享他們的經歷，並給我的劇本貼上標籤，我就知道這個故事引起了共鳴。如果有人和朋友爭論劇本的結局——『主角為什麼要這麼做？ ’——我就知道這引起了他們的思考。最感人的反應是有人説，『謝謝你寫下了我想説但不敢説的話。到。 ’”

**答案品質評估：**
✅ **具體行為** - 明確預期的受眾行為變化
✅ **可觀察性** - 具體的成功指標
✅ **情感共鳴** - 知道什麼反應表明真正的聯繫
✅ **影響力等級** - 從個人到社會的連鎖反應

**關鍵突破：** 將「我希望他們喜歡」轉變為具體的可觀察行為指標。



### **第 6 階段：迭代與反思** — 我實際上該怎麼做？

**原始表面答案：**“我將從構思開始......寫草稿，多次修改......”

**問題診斷：**執行計畫過於模糊，缺乏具體步驟和應對策略，容易導致拖延。

**深入研究過程：**

**🔧 應用框架：[Getting Things Done (GTD)](https://github.com/gtd-methodology/gtd-tools)**

**問題：** 我的第一步是什麼？我今天或明天可以採取的具體行動。

**最初的答案：** 構思，寫草稿。

**GTD 下一步行動定義：**
- **上下文：** @計算機、@安靜的環境、@創作時間
- **所需時間：** 30 分鐘
- **能量等級：** 中等創造力
- **所需工具：** Google Docs、計時器、筆記本

**使用[Task Decomposition](https://github.com/productivity/task-breakdown)進行動作分解：**
1. 開啟 Google 文件（2 分鐘）
2. 設定 30 分鐘計時器（1 分鐘）
3. 寫下三個核心問題（5 分鐘）
4. 自寫答案（20 分鐘）
5. 快速回顧並標記要點（2 分鐘）

**後續問題：** 「構想」太廣泛。今天我要做什麼？我會坐在哪裡？我將使用什麼工具？

**深化答案範例：**
「我今天的第一步是：打開一個新的 Google 文檔，寫下三個核心問題：『這個故事是關於什麼的？』 「主角想要什麼？ 」 「他們最大的恐懼是什麼？ 「我會給自己30分鐘的時間，不停地寫，傾訴所有的想法。然後明天，我將選擇最讓我感動的想法並開始概述第一個場景。我會使用番茄工作法——25 分鐘專注，5 分鐘休息。”

**問題：** 我可能會遇到什麼障礙？如何克服它們？

**深化答案範例：**
「最大的障礙是拖延，我會猶豫是否開始，因為我擔心它不夠好。我的應對策略是：允許自己寫‘垃圾草稿’，初稿不需要完美，它只需要存在。我會把‘劇本寫作’分成小步驟（今天寫人物背景，明天寫對話），這樣我就不會感到不知所措。另外，我會找到一個負責任的伙伴——每週六分享進展，以保持動力。”

**問題：** 經過初步努力，我如何知道哪些有效，哪些無效？

**深化答案範例：**
「完成初稿後，我會給自己一個星期的時間不看它。然後再讀一遍，問：『哪些場景給我帶來情感反應？哪些場景感覺無聊？』保留感性的內容，刪除或重寫乏味的內容。我會與兩三個值得信賴的朋友分享劇本，聽聽他們的真實反應（不是禮貌的談話）。

**答案品質評估：**
✅ **立即行動** - 我今天可以開始的具體步驟
✅ **障礙應急** - 識別主要障礙並制定應對策略
✅ **評估機轉** - 清晰的進度評估方法
✅ **迭代策略** - 知道如何根據回饋進行調整

**關鍵突破：**從「想做」轉變為「準備開始」具體行動狀態。

## 轉化結果：從模糊到清晰的完全變態

**前後比較：**
- **動機層次：**從“表達內在的自我”→“見證痛苦，建立聯繫”
- **受眾層次：**從“年輕人、上班族”→“25歲AE阿信具體形象”
- **方法層面：**從“非線性敍事”→“模擬記憶片段的哲學表達”
- **情感層面：**從“解放、自豪”→“脱掉濕T恤的特定身體感覺”
- **影響力層次：** 從“激發討論”→“具體反饋，例如‘這個角色就是我’”
- **執行級別：**從“構思草案”→“今天打開Google Doc並寫三個問題”

**深化的完整藍圖：**

**為什麼要寫劇本？ **
我寫劇本是因為我記得看完電影後的那個晚上——崩潰。我在螢幕上看到了家庭的裂痕，想到了自己家裡的寒冷。我意識到編劇不僅僅是表達——它是見證——見證我自己和他人的痛苦，然後告訴世界：「我們存在，我們感受到。」如果我不寫，十年後我會後悔把這個故事埋在心裡，成為一個沉默的人。

**我為誰寫作？ **
我是為阿鑫寫的，她是一名 25 歲的 AE，每天加班到晚上 10 點，回到家就精疲力盡。我希望她看完後能想到“別人理解我”，甚至有勇氣做出改變。這個劇本的時機是現在的社會氛圍——經濟不景氣，人工智慧取代人類——讓很多人覺得努力沒有意義。我想提醒大家，即使在荒謬中，我們仍然可以選擇如何生活。

**怎麼寫？ **
我採用非線性敍事，模擬記憶的片段感——觀眾在故事中途了解主角的動機。我有 90 分鐘的限制，迫使我做出選擇：哪些情節點是必不可少的？我有道德底線——不能為了震驚而發布剝削性內容。我希望劇本即使是黑暗的，也能保留人的尊嚴。

**我想要感受什麼？ **
完成後，我想要感受「解放」——就像在炎熱的夏天脱掉汗水浸濕的T卹，肩膀的重量減輕的涼爽。在這個過程中，我願意忍受挫折、自我懷疑、疲勞，因為這些都是創作的一部分。

**我希望觀眾有何反應？ **
我希望他們看完後會想到“我不是唯一一個有這種想法的人”，然後主動與朋友交談，對陌生人更加温柔。如果有人留言“這個角色是我”，我就會知道這是成功的。

**我如何實現它？ **
今天我會打開一個Google Doc，寫下三個核心問題，30分鐘不間斷。明天開始使用番茄工作法勾勒第一個場景的輪廓。我會讓自己“草稿垃圾”，找到一個負責每週進度檢查的合作夥伴。初稿完成後，給它一個星期的時間，然後重新閲讀，刪除無聊的部分，保留情緒化的部分。我會重複至少三遍，直到我認為「這就是我想講的故事」。

## 學習要點及應用指南

**核心學習：**
1. **表面答案辨識** - 學習辨識「安全但無用」的表面答案
2. **深挖技巧** - 掌握從「為什麼」到「具體是什麼」的提問方法
3. **感官描述** - 將抽象概念轉化為可感知的具體體驗
4. **以行動為導向的思維** - 確保每個見解都能轉化為具體行動
5. **框架整合能力** - 靈活組合多種開源工具來實現目標

**套用於您的目標：**

**第 1 步：確定表面答案**
- 您最初的反應是否太“正確”或“安全”？
- 它是否缺乏個人色彩和情感分量？
- 它是否讓您感覺“需要更多思考”而不是“準備好開始”？

**第 2 步：應用深挖技術**
- 使用「為什麼」三層提問法
- 尋找特定的觸發事件或轉折點
- 將抽象概念轉化為感官描述
- 從“每個人”縮小到“一個特定的人”

**第 3 步：建立操作連線**
- 每一個洞察都必須有相應的具體行動
- 設定可觀察的成功指標
- 制定解決主要障礙的策略
- 確保有立即可執行的第一步

**常見陷阱及避免方法：**
- **陷阱 1：滿足於表面答案** → 不斷詢問“是否有更深層的原因？”
- **陷阱二：過度分析而不行動** → 設定時間限制，強調“從80%開始”
- **陷阱 3：目標太宏大** → 分解為今天可以完成的小步驟
- **陷阱 4：忽略情緒層** → 使用身體感覺來描述和驗證答案

**框架的普遍適用性：**
無論您的目標是創業、學習新技能、改善人際關係或個人成長，這個六階段框架都可以幫助您：
- 發現真正的內在動機
- 明確特定目標受眾
- 適合您個性的設計方法
- 預測並準備迎接挑戰
- 建立可持續的執行循環

請記住：好的目標規劃不是一次性的，而是一個不斷深化和調整的過程。當你感到失落或沒有動力時，請回到這個框架，重新審視並深化你的答案。

- **定期清理** - 刪除未使用的工具和進程
- **保持以人為本** - 技術服務於人，而不是相反
- **持續學習** - 隨時了解新工具和方法

請記住：開源框架是工具；真正的力量來自於你的內在動力和持續的行動。選擇適合您的工具組合，建立可持續的改進循環，讓技術成為實現目標的推動者而不是負擔。

import json
from datetime import datetime

class EmotionTracker:
    def __init__(self):
        self.entries = []
    
    def log_emotion(self, story_idea, emotion, intensity, body_sensation):
        """記錄故事想法的情感強度"""
        entry = {
            'timestamp': datetime.now().isoformat(),
            'story_idea': story_idea,
            'emotion': emotion,
            'intensity': intensity,  # 1-10
            'body_sensation': body_sensation,
            'worth_pursuing': intensity >= 7
        }
        self.entries.append(entry)
        return entry
    
    def get_strongest_ideas(self, threshold=7):
        """獲取情感強度最高的故事想法"""
        return [e for e in self.entries if e['intensity'] >= threshold]
    
    def export_to_json(self, filename):
        with open(filename, 'w', encoding='utf-8') as f:
            json.dump(self.entries, f, ensure_ascii=False, indent=2)
